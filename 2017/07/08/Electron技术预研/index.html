IPC通信

主进程和渲染进程通过ipcMain和ipcRenderer两个模块实现通信（类似发布-订阅模式）。

  • ipcMain在主进程中使用，用来处理渲染进程（网页）发送的同步和异步的信息。

  • ipcRenderer在渲染进程中使用，用来发送同步或异步的信息给主进程，也可以用来接收主进程的返回信息。

electron也提供了同步传输的API，但是发送同步消息将会阻塞整个渲染进程，尽量避免。
切忌用 IPC 传递大量的数据，会有很大的性能问题（卡死）。

####remote模块
使用 remote 模块, 你可以调用 main 进程对象的方法, 而不必显式发送进程间消息。

####webContents
webContents负责渲染和控制网页, 是 BrowserWindow 对象的一个属性, 我们使用send方法向渲染器进程发送异步消息。
